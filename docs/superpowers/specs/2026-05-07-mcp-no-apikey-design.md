# MCP Zero-API-Key Design

**Date:** 2026-05-07
**Status:** Approved

## Problem

The `llm-atlas` MCP server and CLI require an external LLM API key (`LLMATLAS_API_KEY`, `ANTHROPIC_API_KEY`, etc.) to generate module summaries. This creates a setup wall for non-technical users ("vibecoders") who want to install and use the tool without configuring API credentials.

The existing approach:
1. AI agent calls `raw_regen` MCP tool
2. MCP server calls `detectLlmConfig()` → reads env vars → calls external LLM API
3. If no key found: generates placeholders or errors

This is backwards — the AI agent that's *already running* should do the thinking.

## Design

### Core Insight

The AI agent (Claude Code, OpenCode, etc.) is itself an LLM. Instead of the MCP server calling a separate API, it should:
1. Provide tools to **read source code** from the project
2. Provide tools to **save generated summaries** to `raw/`
3. The AI agent uses its own intelligence to generate the summaries

No API key. No env vars. The agent itself is the engine.

### MCP Tool Changes

**Removed:**
- `raw_regen` — was calling `runGeneration()` → `detectLlmConfig()` → external LLM

**Added:**

#### `source_read_module`
- **Input:** `moduleName: string`
- **Output:** `{ module, relativePath, files: [{ path, content }], totalChars }`
- Reads all source files for a module (scanner already has this data)
- Validates module name against path traversal
- Returns raw source content for the AI agent to analyze

#### `raw_save_module`
- **Input:** `moduleName: string`, `content: string`
- **Output:** `{ status: "saved", path: "raw/<module>.md" }`
- Writes markdown content to `raw/<module>.md`
- Updates `.raw/.meta.json` with metadata (hash, lastGen, lastCommit)
- Validates module exists in scan result

**Unchanged (keep as-is):**
- `raw_list_modules` — lists all modules with freshness status
- `raw_read_module` — reads existing summary from `raw/`
- `raw_save_module` — full-text search across summaries

### Skill File (`cli/src/commands/init.ts` → generates `.opencode/skills/llm-atlas.md`)

The skill file is the key piece. It teaches the AI agent:

1. What the `raw/` folder contains
2. Which MCP tools are available and what they do
3. **How to generate summaries** — step-by-step workflow
4. The expected summary format (template with Purpose, Key Files table, Data Flow, etc.)
5. Token budget guidance (keep summaries dense, ~800 tokens)
6. **Warning to user** before consuming AI credits
7. Workflow for batch-generating all module summaries

### Init Command Changes

`llm-atlas init` no longer calls `runGeneration()`:

| Before | After |
|--------|-------|
| Create `.rawignore` | Same |
| Create `.raw/config.json` | Same |
| Scan project | Same (show module count) |
| **Run full LLM generation** | **Removed** |
| Install git hook | Same |
| Install/update skill | Same (better content) |
| Update CLAUDE.md | Same |
| Install MCP config | Same |
| Show "Next steps" | Updated: "Ask your AI agent to generate module summaries using MCP tools" |

### CLI Regen Command

`llm-atlas regen` kept for backward compatibility:
- If API key detected: works as before (calls `runGeneration`)
- If no API key: prints worklist (which modules need regen) and tells user to use AI agent

### Files Touched

| File | Action |
|------|--------|
| `cli/src/mcp/server.ts` | Remove `raw_regen`, add `source_read_module` + `raw_save_module` |
| `cli/src/commands/init.ts` | Remove `runGeneration` call, update skill content, update final message |
| `.opencode/skills/llm-atlas.md` | Rewritten as proper workflow skill (generated by init) |
| `cli/src/engine/index.ts` | No changes (kept for CLI `--api` mode) |
| `cli/src/index.ts` | Minor: regen command shows worklist if no API key |

### User Experience Flow

```
1. User: npx @llm-atlas/cli init
   → Config created, skill installed, MCP configured

2. User opens AI agent (Claude Code, OpenCode, etc.)
   → Agent loads .opencode/skills/llm-atlas.md skill

3. Agent reads skill instructions:
   "Use source_read_module to read source code.
    Generate a summary in the specified format.
    Use raw_save_module to save it."

4. Agent: "I found 12 modules. Generating summaries will consume
   AI tokens (~X per module). Continue?"

5. User confirms → Agent generates all summaries using its own LLM

6. Done. No API keys configured.
```
