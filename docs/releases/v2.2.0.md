# v2.2.0 — MCP UX Overhaul

**Tag:** `v2.2.0`
**Date:** 2026-05-08

## Summary

Major UX improvements for AI agents using the LLMAtlas MCP server. Fixes six friction points reported by users: param name mismatches, hidden validation rules, incorrect freshness reporting, empty INDEX.md, no batch workflow, and broken JSONC init. Adds `.mcp.json` support for Cursor/Windsurf/Claude Desktop.

## Changes

### 🛠 MCP Tool Improvements

- **Param aliases:** All tools now accept both `moduleName` (canonical) and `module` (alias). No more `"moduleName must be a non-empty string"` errors when sending `module`.
- **Validation documented:** `raw_save_module` tool description now explicitly lists the 4 required sections. Skill docs include a "Required Sections" table.
- **Test module relaxation:** Modules whose path contains `test/`, `spec/`, or `__tests__/` are exempt from the `## Data Flow` requirement — test fixtures don't have traditional data flow.
- **isFresh fix:** `raw_list_modules` now checks if the summary file actually exists on disk, not just time since lastGen. Modules whose `.md` file is missing from `raw/` are correctly reported as `"stale"`.
- **INDEX.md fix:** Module tree rendering now handles flat module lists (all modules nested, no top-level entries). Falls back to grouping by parent path segment. Shows "(no modules)" instead of blank.
- **`raw_generate_all` (new):** Batch-reads source code for all (or stale) modules at once. Returns full files + exports. Saves N round-trips — AI processes everything in one shot.

### :wrench Init & Config

- **JSONC fix:** `installOpenCodeMcp` rewritten with proper `parseJsonc()` that handles `//` and `/* */` comments. No longer destroys existing MCP config comments.
- **`.mcp.json` support (new):** `llm-atlas init` now creates/updates `.mcp.json` in the project root — used by Claude Desktop, Cursor, Windsurf, and other standard MCP hosts. Format: `{ "mcpServers": { "llm-atlas": { "command": "npx", "args": ["@llm-atlas/cli", "mcp"] } } }`.

### 📖 Documentation

- `README.md` (root): synced with MCP tool updates, param aliases, validation table, batch workflow, staleness file-existence check.
- `cli/README.md`: same updates.
- `.opencode/skills/llm-atlas.md`: parameter names, validation requirements, both workflows (per-model + batch), `raw_generate_all` listing.

### 🔧 Internal

- MCP server version bumped to `2.2.0` (was `1.0.2`).
- Removed orphaned `validateModuleName()` — replaced by `resolveModuleName()`.
- `writeIndexMd()` refactored into 3 rendering modes: hierarchical, flattened-fallback, empty-state.
- All 23 existing tests pass unchanged.

## Upgrading

```bash
npx @llm-atlas/cli@latest init --force
```

This regenerates config files. Existing `raw/` summaries are preserved.
